규칙 기반 검증 시스템입니다.

고정된 명세에 대해 평가합니다.

검증 방식

명세 고정 → 입력 수신 → 비교 → 결과 기록 → 출력 (PASS, FAIL, UNKNOWN)

검증 범위

범위는 규칙으로 비교 가능한 것들입니다.
예: 문서, 소프트웨어, 설정, 스키마, 정책, 배포 산출물.

하지 않는 것

  • 결정
  • 권고
  • 결과 해석
  • 결과 최적화

책임 경계

해석은 외부에 있습니다.
최종 판단은 사용자 또는 조직의 책임입니다.
결과는 권고가 아닙니다.

Rule-based verification system.

Evaluated against fixed specifications.

Verification Method

Specification fixed → Input received → Comparison → Result recorded → Output (PASS, FAIL, UNKNOWN)

Verification Scope

The scope is rule-comparable targets.
Examples: documents, software, configuration, schemas, policies, deployment artifacts.

Does Not Include

  • Decision-making
  • Recommendations
  • Result interpretation
  • Outcome optimization

Responsibility Boundary

Interpretation is external.
Final judgment remains with the user or organization.
Results are not recommendations.
